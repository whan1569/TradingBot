## 데이터 수집 항목 및 출처

### 1. 심리 지표 데이터 (시장 심리 분석)
출처:
- Fear & Greed Index: Alternative.me API (https://alternative.me/crypto/)
- Coinglass: Coinglass API (https://coinglass.com/)
- LunarCrush: LunarCrush API (https://lunarcrush.com/developers)

수집 데이터 항목:
- 공포 & 탐욕 지수 (Fear & Greed Index): 시장 심리 분석
- 롱/숏 비율 (Coinglass, Binance Funding Rate): 숏 강세 vs 롱 강세 판단
- 소셜 미디어 트렌드 (LunarCrush): 트위터, 뉴스에서 언급량 증가 여부

문제점 및 해결법:
- **문제점**: 일부 API는 호출 제한이 존재하여 과도한 요청 시 데이터 수집이 제한될 수 있음.
- **해결법**: 주기적 저장 및 캐싱 전략을 활용하여 API 호출을 최소화하고, 무료 API의 경우 요청 간격을 조정.

### 2. 기술적 분석 데이터 (차트 기반 분석)
출처:
- 바이낸스 REST API: Binance API Docs (https://binance-docs.github.io/apidocs/)
- TradingView API (유료 옵션): TradingView API (https://www.tradingview.com/)

수집 데이터 항목:
- 이동평균선 (SMA, EMA): 단기·중기·장기 추세 확인
- 볼린저 밴드 (Bollinger Bands): 변동성 분석
- RSI (상대강도지수): 과매수·과매도 상태 판단
- MACD (이동평균 수렴·확산): 모멘텀 확인
- 캔들 차트 데이터 (OHLCV): 가격 변동 확인

문제점 및 해결법:
- **문제점**: 고해상도 차트 데이터는 무료 API에서 지원되지 않을 가능성이 높음.
- **해결법**: TradingView 등의 유료 API 활용을 고려하거나, Binance의 WebSocket API를 이용하여 실시간 데이터를 수집 및 저장.

### 3. 과거 매매 데이터 (백테스트 및 전략 평가)
출처:
- 바이낸스 REST API: 주문 이력 (Trade History)
- 바이낸스 WebSocket API: 실시간 체결 정보

수집 데이터 항목:
- 과거 체결 내역 (filled orders)
- 시장가/지정가 주문 기록
- 손절·익절 실행 내역
- 스탑로스, 트레일링 스탑 주문 내역

문제점 및 해결법:
- **문제점**: 바이낸스 API는 일정 기간 이상의 과거 매매 데이터를 제공하지 않을 수 있음.
- **해결법**: 주기적으로 데이터를 저장하여 자체 DB를 구축하고, 장기 데이터를 확보.

### 4. 거래량 및 유동성 데이터 (시장 안정성 분석)
출처:
- 바이낸스 REST API: 거래량 데이터
- 바이낸스 WebSocket API: 오더북 데이터

수집 데이터 항목:
- 거래량 변화 (Volume Analysis)
- 스프레드 (Bid-Ask Spread) 분석
- 유동성 변화 (Liquidity Shifts)
- 주문장 깊이 (Order Book Depth)

문제점 및 해결법:
- **문제점**: 유동성이 낮은 시간대에는 데이터 왜곡이 발생할 수 있음.
- **해결법**: 특정 시간대(예: 미국 시장 개장 전후) 기준으로 데이터를 정규화하여 분석.
