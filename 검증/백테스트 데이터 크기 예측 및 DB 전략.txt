백테스트 데이터 크기 예측 및 DB 전략

=========================================
📌 1. 필요한 데이터 목록
=========================================
1. 바이낸스 거래 데이터
    - 틱 데이터 (초당 수십~수백 건)
    - 1분봉, 5분봉, 1시간봉, 일봉
    - 주문장 (Order Book) 데이터
    - 거래량 및 유동성 데이터

2. 심리 지표 데이터
    - Fear & Greed Index
    - Coinglass (롱·숏 비율, 청산 데이터)
    - LunarCrush (소셜 미디어 데이터)

3. 기술적 분석 데이터
    - 이동평균선, 볼린저밴드, RSI 등

4. 과거 매매 데이터
    - 실제 매매 기록

=========================================
📌 2. 데이터 크기 예상
=========================================
✅ 바이낸스 거래 데이터
    - 틱 데이터: 연간 2~4TB (BTC/USDT 기준)
    - 1분봉: 연간 21MB (BTC/USDT 기준)
    - 주요 심볼 10개 기준
        - 1분봉: 210MB/년
        - 틱 데이터: 20TB/년

✅ 심리 지표 데이터
    - Fear & Greed Index: 73KB/년 (10년 ≈ 1MB)
    - Coinglass 데이터: 8.8MB/년 (10년 ≈ 88MB)
    - LunarCrush (소셜 데이터): 365GB/년 (10년 ≈ 3.65TB)

✅ 기술적 분석 데이터
    - 1분봉 기반 분석: 연간 2GB (10년 ≈ 20GB)

✅ 과거 매매 데이터
    - 1년 ≈ 365MB (10년 ≈ 3.6GB)

=========================================
📌 4. DB 저장 전략
=========================================
✅ 틱 데이터 → 분산 저장 (NoSQL: InfluxDB, ClickHouse, BigTable)
✅ 1분봉 이상 데이터 → MySQL / PostgreSQL (RDS 가능)
✅ 심리 지표 및 매매 데이터 → RDBMS (MySQL, PostgreSQL, TimescaleDB)
✅ LunarCrush 소셜 데이터 → 압축 저장 + 클라우드 (S3, BigQuery 활용)

➡ AWS 설계 시 S3 + RDS + NoSQL 조합이 적절
➡ 데이터 저장 방식 세부적으로 검토 필요
